details batch loading alllows the user to enter data about experimental procedures in bulk the file for submission should be tab delimited with the following headings above each column slidename the slidenames for each experiment being annotated should be entered in the first column expt_type enter a valid experiment type in the second column if you want to create a new experiment type contact the curators at an error occurred while processing this directive tagset a valid procedure must be entered as the third column all the following headings will be interpreted as parameters referring to this procedure until another tagset heading is seen name of parameter this parameter must be a legal option for the previous procedure entered in tagset you can enter multiple parameters as long as they are legal for the preceding procedure tag_some name you can create a user defined parameter by entering it in your batch file with the prefix tag_ in this case it will be entered into the procedure user defined note if the data in your batch file contain any special characters greek letters etc they do not load properly into database such non ascii characters in your data will generate errors and will appear as _ or consider using ascii alternates for example use micro grams or ug or mcg or micro liter or ul or mcl for denoting weight or volume in g or l for denoting temperature
