dataflow online event processing prompt reconstruction calibration overview detector control is that part of the online responsible for controlling and monitoring babar detector hardware archiving ambient data and providing an interface between the data acquisition system and detector hardware via the component proxy in addition detector control is responsible for communicating the status of babar to pep ii and to assist operations by providing coherent integrated access to the status of the detector the detector control infrastructure uses the experimental physics and industrial control system epics which provides an integrated software environment including driver support for hardware readout and control a realtime database for storing and processing data a software bus channel access serving data over a local network and unix client tools for accessing and displaying data the implementation of epics at babar follows the standard application framework for epics safe developed at lbl by steve lewis this page contains links to documentation and other resources associated with online detector control activity there is a hypernews forum for discussion of issues
